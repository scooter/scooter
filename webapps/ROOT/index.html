Scooter Framework Brings Rails to Enterprise J2EE

Summary

A full stack Java web framework that significantly simplifies database-backed web application development.
A Ruby-on-Rails-like framework that is implemented in pure Java.

What does Scooter do better than Rails?

Enterprise Data Access: composite primary key, named sql, views, functions, stored procedures, multiple databases, etc.
Built-in Tools: auto REST, auto CRUD, data browser.

Main Features

Easy to Learn: Scooter only requires you to know:JDK1.4, basic Servlet/JSP. No xml. No tags or annotations to memorize.
No Deployment: No more countless time-consuming deployment cycles during development.
Runtime Configuration: You can change all configuration property files on the fly, including log4j.properties.
Runtime Compilation: Change your Java source files and refresh your browser to see the changes immediately.
Runtime Schema Change: Detect your database schema changes automatically.
Flexible Controller: POJO-like controller, before/after/skip filters, flash and global scopes, pagination and many more.
Active Record: Support belongsTo, hasOne, hasMany, hasManyThrough, polymorphic associations, nested include, cascade, counter cache, actAsTaggable, etc.
Enterprise ORM: Support auto detecting composite primary key, named sql, views, functions, stored procedures, multiple databases, transactions, reference data cache, etc.
Complete I18N: Support internationalization in every layer of the framework.
Web 2.0: AJAX, Markdown.
RESTful Web Services: Automatic RESTify your data store.
Routes: Configurable ways of controlling web access to your data, resources, nested resources.
Many More: Code generator, code statistics, auto CRUD, auto REST, data browser, logging, embedded-web server, etc.
